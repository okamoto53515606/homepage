皆さん、こんにちは。okamoです。

最近、Webメディアを見ていて「疲れるな」と感じることはありませんか？ 画面を覆い尽くすバナー広告、記事の途中で挟まるPR誘導、そしてコメント欄の殺伐とした空気。かつて私たちが夢中で作った「ホームページ」には、もっと純粋な「個人の発信」があったはずです。

そんな想いから、私は新しい個人向けメディアシステム、その名も**「homepage」**を開発しました。

このプロジェクトの最大の特徴は、**開発コードも記事の執筆もAIがサポートする**という点です。私は「企画と指示」だけを行い、実装はすべてAI（Firebase Studio + Gemini）に任せています。

今回は、このシステムの理念的な話だけでなく、**「実際に何ができるのか？」「あなたもすぐに使えるのか？」**という実利面と、具体的な活用法についてご紹介します。

---

## 1. AIが「部下」になる開発体験

まず、開発体制についてです。通常、Webシステムを作るにはエンジニアがコードを書きますが、今回は違います。

私は「こういう機能が欲しい」「デザインはこうして」と指示を出すだけ。AI（Gemini）が裏側でコードを書き、データベースを構築し、デプロイ（公開）までやってくれます。まさに、**AIを優秀なエンジニアとして雇っている**感覚です。

論より証拠、ファイル一覧の右側にあるコメント（コミットメッセージ）にご注目ください。「ありがとう。3ページについては、claudeにテンプレを作成...」「プロンプト履歴。」といった、まるでチャットの履歴のような日本語が並んでいます。

これは私がコードを書いた履歴ではなく、**AIとの対話の記録そのもの**です。私が指示し、AIが生成し、私が「ありがとう」と承認する。このプロセスだけでシステムが出来上がっています。

## 2. 誰でも「フォーク」して、すぐにオーナーになれる

「エンジニアじゃないと作れないんでしょ？」と思われた方、ご安心ください。実はこのシステム、私だけの特注品ではありません。

[Firebase Studio](https://firebase.google.com/docs/studio?hl=ja)を利用し、私が公開している[GitHubレポジトリ](https://github.com/okamoto53515606/homepage)をフォークして取り込めば、**誰でも簡単にこのシステム基盤を手に入れることができます。**

カスタマイズの手順も驚くほどシンプルです。

1.  レポジトリをフォークしてFirebase Studioに取り込む。
2.  AIチャットで「デザインを青ベースにして」「ロゴを変更して」と修正指示を出す。
3.  プレビューで確認し、画面上の**「Publish」ボタンを押す**。

これだけで、修正内容が即座に本番環境へデプロイ（公開）されます。黒い画面でコマンドを叩く必要は一切ありません。

なお、具体的なフォークの手順やFirebase Studioへの取り込み方といった詳細なセットアップ方法については、**後日公開する別の記事にて詳しく解説する予定**です。興味のある方は、ぜひそちらの公開をお待ちください。

## 3. 記事作成もAIが「代筆」

「homepage」がAIに任せているのは、システム開発だけではありません。**記事の執筆（コンテンツ生成）もAIがサポート**してくれます。

「こんなテーマで記事を書いて」「このニュースについて要約して」とAIに依頼すれば、瞬時に下書きを作成してくれます。もちろん、最終的なチェックや修正は人間が行いますが、ゼロから文章をひねり出す苦労はありません。

文章を書くのが苦手な方や、忙しくて時間の取れない専門家でも、AIという「優秀なライター」がいれば、質の高い発信を継続することが可能です。

## 4. 「脱・広告収入」で実現するWin-Winな関係

次に、メディア運営の持続可能性についてです。多くのWebメディアは「広告収入」に依存しているため、PV（ページビュー）を稼ぐために過激なタイトルをつけたり、画面中に広告を貼ったりせざるを得ません。

そこで「homepage」では、**「脱・広告収入モデル」**を掲げ、代わりに**「都度課金」**の仕組みを採用しました。

*   **読者のメリット:** 100円などの少額を決済すれば、「3日間」や「7日間」すべての有料記事が読み放題になります。ノイズとなる広告は一切表示されず、快適に記事に没頭できます。また、サブスクではないため「解約し忘れ」の心配もありません。
*   **運営者のメリット:** PV稼ぎのための大量投稿やSEOハックに追われる必要がなくなります。本当に価値を感じて対価を払ってくれる読者だけに向き合い、質の高いコンテンツを提供することに集中できます。

読者は静かな環境で良質な情報を得られ、運営者は正当な対価を得る。このシステムは、双方が幸せになれる**Win-Winの関係**を目指しています。

決済システムにはStripeを採用しており、インボイス対応の領収書メール自動送信なども完備。これもAIに「Stripe入れて、領収書も出るようにして」と頼んだだけで実装されました。

## 5. 「透明性」で守るコメント欄

Webメディアの悩みの種である「荒れるコメント欄」や「世論操作」に対しても、実利的な機能で対抗策を講じています。

*   **国・地域の表示:** コメント投稿者の接続元（例：JP / 東京都）を表示。
*   **日替わりID:** 2ch（5ch）のように、その日だけのIDを表示。

これにより、「誰が（どこから）言っているか」が可視化されます。組織的な工作や、botによる大量投稿があれば一目でわかるため、結果として**健全な議論の場**が保たれやすくなります。

## 6. 活用イメージ（こんな使い方ができます）

では、この「homepage」システムは具体的にどのような場面で使えるのでしょうか。

### ケースA：専門家のニュースレター
特定の業界（医療、法律、技術など）の深い知見を持つ方が、AIに執筆を補助させながら効率的に情報を発信。広告収入に頼る必要がないため、大衆受けを狙う必要がなく、**「質の高い情報を、必要な人だけに届ける」**ことが可能です。

### ケースB：クリエイターの制作日誌
作品のファンに向けて、制作の裏側やラフスケッチを公開する場として。Markdownでシンプルに書けるので、画像とテキストをポンポン置いていくだけ。余計な広告バナーがない分、作品そのものの世界観を壊さずに伝えられます。

### ケースC：期間限定プロジェクトの広報
イベント期間中だけ情報を集中的に発信し、終了後はそのままアーカイブとして残す。サブスク課金ではないので、更新が止まっても「金だけ取り続けている」と批判されるリスクがありません。
